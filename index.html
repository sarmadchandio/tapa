About this comparison. We extracted 100 stop clips per debate (Trump only, ±50 ms / +100 ms context around each stop from TAPA) for 3 debates and ran both TAPA's VOT (CMUdict proportional timing + Praat intensity peak) and Dr.VOT (BiLSTM + structured-SVM, GPU-accelerated). Dr.VOT reports signed VOT (negative = prevoicing). TAPA's VOT here is really a phone-boundary offset — not a true burst-voicing interval — because CMUdict proportional timing lacks the acoustic landmark detection that would produce meaningful VOT. That shows up as near-zero TAPA values below.

Does MFA alignment rescue TAPA's VOT? Tab 3d showed that TAPA's CMUdict-timed VOT clusters near 0 ms and disagrees with Dr.VOT. This tab re-runs the same TAPA VOT algorithm (Praat intensity peak for burst + pitch tracker for voicing onset) but with Montreal Forced Aligner phone boundaries instead of CMUdict proportional timing. If CMUdict coarseness were the problem, MFA should bring TAPA into agreement with Dr.VOT.

Finding: MFA boundaries don't rescue TAPA. "MFA-TAPA" VOT still clusters at ~0 ms, and the correlation with Dr.VOT is essentially zero across all phones. The underlying issue is the VOT algorithm, not the alignment — Praat's pitch tracker finds voicing immediately at the stop-vowel boundary regardless of how that boundary is placed, so VOT collapses to ~0 for every clip. A burst-aware algorithm (or an end-to-end model like Dr.VOT) is required.
